the Singaporean Tech savvy community

Inspiration

The Guardian

Singapore Tech Today's web design, user interface, user experience is inspired by british news website: The guardian, its font, primary and secondary color provides a sense of security, reliability, prestige and brand identity of STT

The Verge

Singapore Tech Today's writing style, jounalism and content are inspired by tech news outlet: The verge. STT's containerized content is a inspired by the verge's neat, stacking layers of content and headlines, forming a unique, metro-style and techy feel

MarketWatch and Yahoo Finance

Singapore Tech Today's other elements like the Live blinker and stock ticker are inspired from Yahoo Finance and MarketWatch. Which i feel makes the website feels lively, updated and constant

Accessibility

User Interface

Inspired by British Newsletter publisher - the guardian, Singapore Tech Today's user Interface feature a vast array of content while retaining form, factor and recheablity

1. RECHEABILITY

elements such as the taskbar, footer and itemized containers makes for easy viewing, navigation and mobile reachability

2. RESPONSIVENESS

Site was designed with multiple screen sizes in mind, never will a element be overflowed, missing or juttering out of its container at any screen size

3. FAVEICONS AND TITLE

Faveicons and titlenames allow for users to quickly navigate to the webpage should they have multiple tabs opened

User Experience

Inspired by British Newsletter publisher - the guardian, Singapore Tech Today's user experience feature a vast array of content whilst avoiding overcluttering of on screen elements and objects

1. COLOUR

The deep blue #040362 as the main colour of the Newsletter provide a sense of security and reliability, letting our readers know that the content can be trusted, vetted and reliable

2. SIMPLE AND ROBUST TYPEFACE

STT's feature a a typeface that is a cousin to new times roman: Inria, a sense of prestige and elegance. The lack of necessity to implement alternate fonts as new times roman, default html font closely resembles Inria: this reduces code maintanibility retains portability

3. DYNAMIC CONTENT

Animation in the stock tickers, subscribe page, and live blinker and video in the log-in page helps provide a sense of liveliness to the webpage, make it seem more dynamic and interesting Document Object Model DOM manipulation also allowed for dynamic content

Functionality

Singapore Tech Today's website is packed with content and features

1. Ease of access for content creators

With mustache's logicless templating, content creators can easily update, delete, or post new content with minimal technical skills

2. Less resource intense

The website features DOM manipulation in the subscribe and log-in page which negates the necessity of having an extra page to create an account, this reduces the need for the browser to displayer yet another page, saving on the users bandwidth and computing resources

Usability

Mustache Templating

With Mustache Templating, journalists can easily add in, remove, update content easy through updating the JSON data format. Moreover, Mustache is logic-less, make it less complicated for non-developers to change content information, this reduces time, complexity and even labour cost, eliminating the need developers to update content for the writers

No Phantom Links

The lack of phantom links means every element on my website is usable, and clickable, all the elements on the graphical interface are done with functionality as the focus

Simple, robust, human centric UX and UI

As mentioned in the Accessibility section, the UX and UI are throughly have been researched, and desgined to be as simple to use as possibe, while packing as much content as possible, if the Inria font fails, the Default html New Times Roman font servers as a similar alternate backup font


    Learning

Advance Javascript

i have learnt advance javascript techniques such as mustache templating, document object model DOM manipulation. These technique allowed me to creaete dynamic, interesting, and functional user interface and experience, whilst reducing the complexity for code maintance and makes it much easier for Journalist, content creators and administrators to update, add, or delete content such as newsarticles
I was able to complete coding using these advance Javascript code by reading online materials, watching the coursera videos, and looking for examples on W3 schools.
I then implement the "bare-bones" version of my code into my code editor, and try to adjust the code such that it will be compatible with my idea styling and functionality

CSS frameworks, styling and HTML

I have learnt to utilize both self-desgined CSS styles in conjunction with CSS frameworks like BOOTSTRAP to align, position, color content in my website. I also used CSS to create animated elements such as the Live Blinker, the animated images in the subscribe page. Extensive use of media queries in the CSS allows for responsiveness across a wide range of devices and screen sizes'
I was able to achieve this through learning about styling methods through W3 schools. Many common methods such as accordion and modal were learnt from W3 school's own website. In addition, i have also look at the reading resource provided on coursera and the videos do help in getting a better understand and feel of what CSS code do i require

Debugging Skills

I have learnt debugging skills for HTML, CSS and Javascript. I have learnt how to navigate the browser console, to find conflicitng styling code in CSS, incompatible HTML code.
I have also learnt how to utilize console.log in different functions for different variables to debug for Javascript related issues
For CSS debugging, i was able to sucessfully debug just by observing the browser console. I was able to find out conflicing classes and styles from bootstrap that were affecting my non-bootstrap through my browser's console.
As for how i was able to successfully debug the Javascript aspect of my website. It all comes down to the debugging skills i obtained from Introduction to programming 1, where the coursera videos, weekly tips on debugging has helped be tremendously in effective and effecient Javascript Debugging

Evaluation 1

Successful Aspects of my work

Easy to understand code

Neat and tidy code, with proper indentations and comments allow for any developers to know the structure of the website, and their inner workings Proper usage of directories, sorting HTML CSS and javascript files into different folders named after their functionality. Proper naming of classes, IDs and usage of HTML tags allow for any developer to comprehend the code they are looking at. All these practices reduce the resources required should a unexpected issue occurs, and where developers have to debug code. A tidy, neat and easy to understand code, allows for time to be saved from debugging, and be utilized on more productive and important aspect of this project The proper use of class naming has allow for easy finding of classes due to its easy to remember nature, and how easy it is to find in a complex code base through cltr F or find and replace

Inbuilt redundacies

Alternatively hosting a backup sites at Github pages, using pull and push to store code, making it extremely portal. Even this report. contains several redundacies including backup iframe links to a alternative report should this somehow fail, using git as a form of version control also enables me to explore multiple design choices to choose from. By having redundacies reduces the risks of total loss of data, service and potentially even revenue in-flow. Technical glitches, bugs and unexpected erros are certain to happen when it comes to any software or hardware. Having redundacies reduces the severity of such incidents

Unit Testing

Unit testing allows me to test my website for any bugs, making a more reliable and user friendly product. Unit testing ensure that the user would have a great experience using this website increasing the probability of subscriber count, making me more money. Additionally, unit testing also ensure that there are no security risks, protecting both the user and the provider from cyber related threats and exploits I tested this website everything i make a commit to the code base to ensure nothing breaks and bugs are addressed as soon as possible. This is to anticipate for edge case user behaviours, such as users clicking on pages in different orders. Sometimes i do uncover issues like this leading to an error 405 or 404 not found, indicating i have linked a a-tag inapproperiately

What aspects of your work could be improved? How might you do things differently another time?

    More Dynamic Content

    Even though dynamic content are implemented in this project, my article page still suffers from the lacks dynamic content like more images, or slideshow containers to show more related images, which would had create a sense of "liveliness" and keep user focus and interest. Right now i only have the stock ticker, some animations and videos to provide that sense of liveliness that would be good for my website. The next time, i would manage my time well, hone my CSS skills, to implements these improvements

    Some pages on my website do not resemble my wireframe

    Due to time and workload constraints, i was not able to replicate fully how my wireframe is, which requires more extensive coding
    also, using DOM elements has altered and made me realize that my wireframe design might not be the most practical, this is because i will slowly make adjustments through unit testing to slowly improve the user interface to improve the overall user experience. The next time, i would utilize my UX and UI skills and knowledge to create a design that users would like and be practical, before i start coding for the website

    Utilizing more CSS frameworks

    I only manage to utilize bootstrap CSS for one part of my navigation bar,
    this stems from my limited understand of bootstrap classes, which discouraged me to utilize bootstrap more on my website. By utilizing more CSS bootstrap, i can save more time when it comes to coding for responsivness for this webpage. This is because most bootstrap templates as given in the bootstrap documentations do have in-built responsivness. The time saved from utilizing bootrap could have been used on other aspects of this project such as the reasons mentioned above Next time, with more practice and in-depth studying on the bootstrap documents, to with lesser time, utilize the bootstrap responsiveness features
